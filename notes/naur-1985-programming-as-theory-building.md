# naur-1985-programming-as-theory-building

Peter Naur, "Programming as Theory Building," *Microprocessing and Microprogramming* 15 (1985), pp. 253–261. A short paper (~14 pages) with outsized influence on software engineering epistemology.

Naur argues that the programming activity is fundamentally about building a theory — in Ryle's sense — of the relationship between a program and the real-world affairs it handles. The theory is not reducible to the source text, the documentation, or any formal specification. It is a cognitive capacity held by the programmers who built it.

The paper develops several consequences: that source text underdetermines meaning, that program revival by newcomers systematically degrades design quality, that documentation cannot substitute for the held theory, and that programming education should develop theory-building capacity rather than text-writing technique.

# Task: static thread pool decouples neural inference from the audio callback to ensure real-time safety

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

Core architectural decision in anira. The InferenceManager, SessionElement, and static ThreadPool together separate inference from the audio callback:

- **Static design:** prevents oversubscription (multiple instances each creating high-priority inference threads beyond available hardware threads). The pool is static — created once, shared across instances.
- **ThreadSafeStructs:** two synchronization strategies available:
  - Atomics (std::atomic<bool> / std::binary_semaphore for data access; std::atomic<int> / std::counting_semaphore for task submission): strict RT-safe, no syscalls
  - Semaphores (std::counting_semaphore::try_acquire_until): allows controlled blocking in the audio callback to receive processed data sooner, reducing latency at the cost of minor syscalls
- **Parallel inference:** for stateless models, multiple threads can run inference simultaneously, enabling throughput gains
- **Verified:** RadSan testing of the anira library's process method confirms zero RT violations

The InferenceManager handles buffer size alignment (Letz algorithm) and latency calculation. The SessionElement stores audio data and inference results. Timestamps via buffer counting ensure output sequence correctness.

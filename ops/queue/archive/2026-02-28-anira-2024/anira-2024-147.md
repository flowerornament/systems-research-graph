# Task: host buffer to model input mismatch and parallel inference remain unbenchmarked in real-time audio contexts

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

Two explicit limitations acknowledged by the authors in the Discussion section:

**1. Host buffer / model input size mismatch:**
The benchmarks deliberately aligned model input size with host buffer size to isolate the buffer-size variable cleanly. But real-world audio applications frequently present varying host buffer sizes that differ from the model's fixed input shape. This is precisely the scenario anira's Letz-algorithm-based H_adapt latency term addresses architecturally. The performance behavior under mismatch — how much overhead the alignment buffering adds, how it interacts with the inference time distribution — remains unmeasured.

**2. Parallel inference:**
Anira supports parallel inference for stateless models via the multi-threaded ThreadPool. For a single instance with multiple threads, the inference throughput could be substantially different from single-threaded benchmarks. This was not measured. The ThreadPool's static design (preventing oversubscription) is presumably beneficial for parallel configurations, but the actual throughput/latency trade-off at different thread counts is unknown.

Both gaps are consequential for murail's deployment context, where the host buffer size is determined by the audio driver and may not match the trained model's input shape.

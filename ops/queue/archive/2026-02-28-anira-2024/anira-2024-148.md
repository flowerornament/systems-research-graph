# Task: tensorflow lite outperforms libtorch for small cnn models but libtorch becomes faster as model size grows

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

Follow-up study LMM-III comparing CNN-1k (931 params), CNN-15k (15,300 params), and CNN-29k (29,669 params) across ONNX Runtime, LibTorch, and TensorFlow Lite (DS-III, N=108,000):

- CNN-29k: TFLite significantly inferior to LibTorch (p < 0.0001)
- CNN-15k: crossing region — results become less consistent
- CNN-1k: TFLite no longer significantly different from ONNX Runtime; LibTorch becomes the slowest

ONNX Runtime is fastest across all CNN sizes in this comparison.

The crossing behavior is significant for model selection: engine choice interacts with model size. A lightweight neural audio effect (~1k params) should prefer TFLite or ONNX Runtime over LibTorch. A full-size audio effect model (~30k params) should prefer ONNX Runtime and can use LibTorch but should avoid TFLite.

The mechanism is not explained by the paper, but likely reflects: TFLite's inference overhead is lower for small models (simpler runtime, fewer abstractions); LibTorch's runtime investment pays off as computational complexity grows.

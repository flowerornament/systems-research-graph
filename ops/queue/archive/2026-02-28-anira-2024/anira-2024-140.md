# Task: inference engines violate real-time principles on every inference not just initial ones

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

The Real-time Sanitizer (RadSan) tests show persistent real-time violations across ALL inference engines, ALL models, and ALL inference counts. The violation graphs never reach zero. This contradicts Stefani et al. 2022 [12], which claimed that engines "consistently ensure real-time safety after the initial inference" -- a conclusion attributed to Stefani's use of a Xenomai Cobalt hard-RT kernel rather than intercepting system library calls.

Specific violations observed:
- LibTorch: highest violations, especially on initial inferences; malloc, free, calloc, pthread_mutex_lock, pthread_rwlock_rdlock; CNN-29k also uses sleep and fopen during initial inference
- TensorFlow Lite: exclusively memory operations (malloc, free, aligned_alloc); RNN-2k shows consistently high violations across all inferences; CNN-29k and HNN-11k show minimal memory activity
- ONNX Runtime: HNN-11k shows consistently intensive memory usage; CNN-29k fewer violations; malloc, free, posix_memalign
- anira library itself: NO violations detected in the audio callback

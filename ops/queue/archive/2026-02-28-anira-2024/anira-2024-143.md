# Task: onnx runtime is fastest for stateless neural models while libtorch is fastest for stateful models

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

Statistical findings from LMM-I (stateless: CNN-29k, HNN-11k) and LMM-II (stateful: RNN-2k, CNN-29k, HNN-11k, excluding ONNX Runtime):

**Stateless models (LMM-I):**
- Speed order: Bypass > ONNX Runtime > LibTorch > TensorFlow Lite
- ONNX Runtime consistently outperforms LibTorch and TensorFlow Lite for CNN-29k and HNN-11k (all comparisons p < 0.0001, except ONNX vs TFLite for HNN-11k at p = 0.045)

**Stateful models (LMM-II):**
- ONNX Runtime excluded (does not support stateful LSTM operations)
- LibTorch outperforms TensorFlow Lite on average
- But TFLite vs LibTorch results are inconsistent across models: TFLite is slower for CNN-29k and RNN-2k, but faster for HNN-11k

The inconsistency between TFLite and LibTorch motivates the follow-up CNN size study (LMM-III), which reveals the crossing point is model size: TFLite is faster for small CNNs, LibTorch becomes faster as model grows.

Note: ONNX Runtime does not natively support stateful operations (retaining LSTM state between calls), confirmed by the ONNX Runtime architecture documentation. The RNN-2k model uses a stateful LSTM layer that must update its internal state after each buffer — this is architecturally incompatible with ONNX Runtime's stateless operation model.

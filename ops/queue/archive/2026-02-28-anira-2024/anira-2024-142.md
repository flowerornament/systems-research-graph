# Task: anira latency formula derives minimum required buffering from worst-case inference time and buffer size mismatch

## Source Context

From [[anira-2024]] (Ackva & Schulz, TU Berlin, arXiv:2506.12665v1):

Two formal equations establish the minimum latency required for continuous signal flow:

**Equation 1 (baseline):**
```
L_total = H_adapt + ceil(I_max / H_host) * H_host + M_int
```
Where:
- H_adapt = latency to adapt buffer sizes (from Letz algorithm for mismatched sizes)
- I_max = combined worst-case inference time in samples
- H_host = host buffer size
- M_int = internal latency of the model

The ceil() ensures I_max rounds up to next integer multiple of H_host because inference data is requested only once per audio callback. The second term is always >= H_host, so L_total > 0 always.

**Equation 2 (with semaphore wait time):**
```
L_total = H_adapt + ceil((I_max - H_host * W) / H_host) * H_host + M_int
```
Where W = proportional wait time (0.0 to 0.95). Setting W > 0 introduces controlled blocking in the audio callback (requires semaphore-based ThreadSafeStructs), enabling zero added latency for some configurations.

Buffer size mismatch is handled via the Letz algorithm: analyzing mod(nA, B) until pattern repeats to find minimum H_adapt for buffers of size A and B.

The worst-case inference time I_max cannot be derived analytically — it must be measured empirically via benchmarking, because inference engines have non-deterministic runtimes.
